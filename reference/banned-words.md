# Banned Words and Phrases (Living Reference)

Source material: CFPB consent orders, FTC enforcement actions, FDIC Part 328, Reg DD, FINRA Rule 2210. Update when new enforcement clarifies a phrase.

Use this as the first pass on every consumer-facing asset. If a phrase appears in the left column, it is presumed banned unless the right-column conditions are met.

| Banned phrase | Why / safer alternative |
|---|---|
| "Free" (standalone) | Implies no cost ever. Safer: "$0 monthly fee" + footnote listing every condition that has a fee (out-of-network ATM, foreign transaction, expedited delivery, paper statements, overdraft). |
| "No fees" | Almost never literally true. Safer: "No monthly maintenance fee" + specific list of fees that do exist. |
| "0% fees forever" / "fees never" | "Forever" claims trigger UDAAP because future fee changes are predictable. Safer: "$0 monthly fee" with a "fees subject to change with notice" clause. |
| "Guaranteed" returns / yield / earnings | Reg DD and UDAAP. Deposit account yields are variable. Safer: "[X]% APY as of [date]. Variable rate." |
| "Risk-free" | Almost never usable. Even FDIC-insured deposits carry risks (the fintech failing in a non-pass-through structure, fees reducing principal). Safer: state the actual coverage. |
| "FDIC insured" (without partner bank) | FDIC Part 328 prohibits non-banks from claiming FDIC status as their own. Safer: "Banking services provided by [Partner Bank Name], Member FDIC." |
| "We are FDIC insured" / "[Brand] is FDIC insured" | Same as above. The fintech is not the insured entity. Safer: see above. |
| "Your money is FDIC protected" (used by non-bank) | Implies fintech-level coverage. Safer: "Your deposits are held at [Bank], Member FDIC, where they are eligible for FDIC insurance up to $250,000 per depositor, per insured bank, per ownership category." |
| "Instant" transfer / "instant" payout | Only usable if technically true 100% of the time. Sendwave/Chime October 2023 was triggered by "instant" claims that failed under load. Safer: "Most transfers complete in under [X] minutes" + state the exception window. |
| "Best APY" / "highest APY" | Reg DD requires effective date and methodology for any superlative comparison. Safer: "[X]% APY as of [date], among the top rates in [category] per [source, date]." |
| "Highest yield" | Same as above. |
| "High yield" (no APY shown) | Reg DD requires APY when discussing a savings rate. Safer: "[X]% APY" stated explicitly. |
| "Earn up to X%" (no conditions) | Reg DD requires conditions to reach the top tier. Safer: "Earn up to [X]% APY. Top tier requires balance of $[Y] and [other condition]." |
| "Save up to $X" (no methodology) | UDAAP and FTC substantiation. Safer: show the math. "Customers saved an average of $[X] in fees in [period] vs [comparable account at named competitor]. Methodology: [link]." |
| "Get rich" / "build wealth fast" | Wealth claims on deposit products are deceptive. Safer: stick to the actual feature. |
| "Pays you to bank" | Implies guaranteed earnings. Safer: state the rate or rewards structure with conditions. |
| "Bank-grade security" | Vague and often inaccurate when the fintech relies on partner-bank security plus its own stack. Safer: name specific certifications (SOC 2 Type II, ISO 27001) and link to the security page. |
| "Military-grade encryption" | Marketing fluff with no defined standard. Safer: "256-bit AES encryption in transit and at rest." |
| "100% safe" / "completely safe" | Absolute safety claims fail UDAAP. Safer: describe specific protections (FDIC pass-through up to $250K, fraud monitoring, customer reimbursement policy with terms). |
| "Money you can trust" | Vague claim of safety; fails substantiation. Safer: name the partner bank, name the certifications, link the disclosures. |
| "Beats your bank" | Comparison without source and date. Safer: "[X]% APY vs the [Y]% national average per FDIC weekly rate as of [date]." |
| "Trusted by millions" (no source) | Substantiation required. Safer: "[N] customers as of [date]" with the count tied to active users, not lifetime signups. |
| "Award-winning" (no specifics) | Vague unless cited. Safer: "[Award], [year], by [organization]." |
| "Built for you" / "designed for you" | Fluff, not banned, flag for editing. |
| "Switch in seconds" / "open in seconds" | Often inaccurate. KYC, funding, and partner-bank checks add minutes or hours. Safer: "Most accounts are opened in under [X] minutes" + condition. |
| "Approved instantly" | Same. KYC sometimes requires manual review. Safer: "Most applicants are approved in under [X] minutes." |
| "No credit check" (when there is any) | If a soft inquiry occurs, the claim is misleading. Safer: "No hard credit check. We may run a soft inquiry that does not affect your credit score." |
| "Guaranteed approval" | Banned. Approval is conditional on KYC, funding source, and underwriting. Safer: omit. |
| "Pre-approved" | Heavily restricted. ECOA and Reg B rules apply. Safer: "Eligible for [product] based on [criteria]." |
| "Lowest fees" / "cheapest" | Comparison superlative requires source and date. Safer: cite the comparison with date. |
| "100% transparent" | Fluff plus risk: implies disclosure of everything, then any non-disclosure becomes a UDAAP exposure. Safer: link the fee schedule and disclosures. |
| "Always-on support" / "24/7 support" | If support has any limit, this is misleading. Safer: state actual hours and channels. |
| "Hidden fees we don't have" | Comparative fee claims require substantiation. Safer: state your fees, link the schedule, do not characterize competitors without dated sources. |
| "Better than a bank" | Often false; the fintech is not a bank. Safer: state specific differences (e.g. higher APY at specific rate, lower specific fee). |
| "Your money grows" | Implies guaranteed return. Safer: "Earn [X]% APY as of [date], variable." |
| "Cash back guaranteed" | If conditioned on category or merchant, "guaranteed" is misleading. Safer: "[X]% cash back on [eligible categories] up to $[Y] per [period]. Conditions apply." |
| "Up to $X cash back" | Requires the conditions to reach the cap. Safer: state the cap, the conditions, and the exclusions. |
| "Build credit guaranteed" | Banned. Credit-building is conditional on bureau reporting and customer behavior. Safer: "Reports payment history to [bureaus]. Results vary." |
| "Investment" / "investing" applied to a deposit account | Reg DD and SEC concepts. Deposits are not investments. Safer: "savings account" or "deposit account." |
| "Yield" alone (instead of APY) | Reg DD requires APY. Safer: "[X]% APY." |
| "Get paid early" without conditions | Common neobank claim. Conditions: depends on employer's payroll file submission. Safer: "Get paid up to 2 days early when your employer submits payroll early. Timing is not guaranteed." |
| "Overdraft-free" | If overdraft is just renamed (cash advance, paycheck advance), the claim becomes UDAAP exposure. Safer: name the product correctly and disclose fees. |
| "Free money" / "easy money" | Banned across the board. |
| "Limited time" / "ends soon" without truth | False scarcity is a dark pattern. Safer: only used when there is a real, dated promo end. |
| "Only N spots left" without truth | Same. Safer: only with real inventory. |

## Update protocol

When a CFPB or FTC action introduces a new banned phrase pattern, append a row here with the citation. Do not delete rows; if a phrase becomes safer over time, annotate the row with the new context rather than removing it.
